System Architecture

Three-tier architecture for scalability, performance, and maintainability

Figure 1: System Architecture - Three-tier design with React frontend, FastAPI backend, and MongoDB database

Architecture Layers

1

Presentation Layer (Frontend)

React Single Page Application (SPA) providing responsive, device-agnostic user experience.

  • React 19.0: Component-based UI with virtual DOM
  • Context API: State management for user sessions
  • Chart.js: Interactive data visualizations
  • Responsive Design: Mobile-first approach
2

Application Layer (Backend)

FastAPI server hosting business logic, risk assessment algorithm, and RESTful endpoints.

  • FastAPI: High-performance async framework
  • Risk Algorithm: Multi-factor weighted scoring
  • Async/Await: Non-blocking concurrent requests
  • Pydantic: Data validation and serialization
3

Data Layer (Database)

MongoDB document store with flexible schema for evolving student metrics.

  • MongoDB: NoSQL document database
  • Collections: Students, Financials, Housing, Enrollments, Courses, Alerts
  • Aggregation Pipelines: Efficient data reconstruction
  • Student_ID: Primary linking key across entities
