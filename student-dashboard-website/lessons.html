Lessons Learned & Team Reflection

Key insights from developing the Centralized Student Dashboard

1

Data Integration is Time-Intensive

Challenge: Initial project timelines were overly optimistic regarding data cleaning and schema design across multiple derived sources. Lesson: Robust pre-processing and ensuring referential integrity are high-risk early phase activities requiring significant time allocation. Future projects should allocate 30-40% of development time to data integration alone.

2

Prioritizing Value Over Features

Challenge: The team initially planned extensive automation features that would have delayed delivery. Lesson: We learned to strictly scope the prototype to highest-value features (risk identification and profile viewing), deferring complex automation to the next phase. This demonstrated the importance of delivering a Minimum Viable Product (MVP) focused on solving the core problem: timely intervention.

3

Technology Choice Matters

Challenge: The initial plan used Django/PostgreSQL, but performance concerns led to a mid-project pivot. Lesson: The decision to switch to React/FastAPI/MongoDB, while introducing a learning curve, ultimately provided a more scalable, flexible, and portfolio-ready product. Early technology evaluation and willingness to adapt are crucial for long-term success.

4

Iterative Testing Reveals Hidden Issues

Challenge: Several edge cases in risk calculation only emerged during user testing with synthetic data. Lesson: Continuous testing with diverse data scenarios is essential. Regular stakeholder feedback sessions helped identify usability improvements that wouldn't have been apparent from internal testing alone.

5

Documentation is Investment, Not Overhead

Challenge: Early documentation was deprioritized to focus on coding, causing confusion during integration. Lesson: Comprehensive documentation (API contracts, data models, architecture diagrams) significantly accelerates development velocity and team coordination. Time spent documenting is time saved debugging.

Next Steps & Future Enhancements

  • Production Security: Implement JWT token authentication, password hashing (bcrypt), and role-based access control for secure deployment
  • Machine Learning Integration: Replace static weighted algorithm with dynamic ML model (Logistic Regression or Random Forest) trained on historical retention data
  • Action Logging: Build intervention tracking functionality allowing advisors to log outreach attempts, meetings, and student progress
  • Mobile App Development: Create native mobile applications for advisors to access student profiles and receive real-time alerts
  • Integration with LMS: Connect with Learning Management Systems (Canvas, Blackboard) for real-time assignment completion and engagement data
  • Predictive Alerts: Automated early warning notifications to advisors when student risk scores cross critical thresholds
  • Longitudinal Studies: Track cohort outcomes over multiple semesters to validate and refine the risk algorithm
